# 变更记录规范

当前仓库使用 `__LAYOUT_LABEL__`。

生效 changelog 路径：

- `__CHANGELOG_PATH__`

`repo-check.toml` 里的 `changelog_path` 是提交阶段的实际 source of truth，不只是展示字段。

## 基本要求

- 有实际改动时，必须同步更新当前 layout 对应的 changelog。
- 不能只改 changelog 而没有实际改动。
- 默认不允许删除当前仍然活跃的 changelog。
- 如果删除的是一个已经整体删除的 crate，对应的 `crates/<name>/CHANGELOG.md` 可以随 crate 一起删除。

## 只允许修改 `[Unreleased]`

默认情况下，只允许改 `[Unreleased]`。

如果正在切 release，确实需要回写已发布段落，可以显式使用：

```bash
OMNE_ALLOW_CHANGELOG_RELEASE_EDIT=1 git commit ...
```

## layout 语义

如果 layout 是 `root`：

- 只能使用根级 `CHANGELOG.md`

如果 layout 是 `crate`：

- 每个发生实际变更的 `crates/<name>/` 都必须更新自己的 `crates/<name>/CHANGELOG.md`
- root 级治理代码、文档和其他不属于活跃 crate 的改动，统一归属到 `__CHANGELOG_PATH__`
- `crates/<name>/` 下只有真实 crate 才会成为独立 changelog owner；共享目录不会自动变成 crate
- 根级 `CHANGELOG.md` 不是当前仓库提交阶段的 changelog 入口，除非它正好就是 `repo-check.toml` 里配置的 `changelog_path`
