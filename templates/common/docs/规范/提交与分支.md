# 提交与分支规范

## 分支命名

默认允许：

- `main`
- `master`
- `feat/*`
- `fix/*`
- `docs/*`
- `refactor/*`
- `perf/*`
- `test/*`
- `chore/*`
- `build/*`
- `ci/*`
- `revert/*`

## 提交消息

默认要求 Conventional Commits：

```text
<type>(<scope>)!: <subject>
```

允许的 `type`：

- `feat`
- `fix`
- `docs`
- `refactor`
- `perf`
- `test`
- `chore`
- `build`
- `ci`
- `revert`

允许直接放行的特殊提交：

- `Merge ...`
- `Revert "..."`
- `fixup! ...`
- `squash! ...`

## breaking change 规则

如果 `__PACKAGE_MANIFEST_PATH__` 或同类 manifest 发生了非 `0` 大版本 major bump：

- `pre-commit` 默认拒绝，需要显式设置 `OMNE_ALLOW_MAJOR_VERSION_BUMP=1`
- `commit-msg` 还要求提交消息显式声明 breaking change：
  - 标题带 `!`
  - 或 footer 使用 `BREAKING CHANGE:` / `BREAKING-CHANGE:`

这条规则当前只根据 manifest 的版本号变化判断，不自动推断运行时兼容性。
