# 提交与分支规范

## 分支命名

默认允许：

- `main`
- `master`
- `feat/*`
- `fix/*`
- `docs/*`
- `refactor/*`
- `perf/*`
- `test/*`
- `chore/*`
- `build/*`
- `ci/*`
- `revert/*`

## 主分支保护

`main` 应启用受保护分支规则。

合并前至少要求：

- 必需的 CI / CD status checks 全部通过
- 不绕过保护直接 push 到 `main`
- 用和默认分支一致的 CI 入口验证待合并改动

如果当前仓库直接使用模板自带的 `.github/workflows/ci.yml`，那么远端 required checks 应明确选中它暴露的 job context：`test` 与 `clippy`。

本地 hook 只负责把问题前移到提交阶段，不替代远端分支保护。

## 提交消息

默认要求 Conventional Commits：

```text
<type>(<scope>)!: <subject>
```

允许的 `type`：

- `feat`
- `fix`
- `docs`
- `refactor`
- `perf`
- `test`
- `chore`
- `build`
- `ci`
- `revert`

允许直接放行的特殊提交：

- `Merge ...`
- `Revert "..."`
- `fixup! ...`
- `squash! ...`

## breaking change 规则

如果 `__PACKAGE_MANIFEST_PATH__` 或同类 manifest 发生了非 `0` 大版本 major bump：

- `pre-commit` 默认拒绝，需要显式设置 `OMNE_ALLOW_MAJOR_VERSION_BUMP=1`
- `commit-msg` 还要求提交消息显式声明 breaking change：
  - 标题带 `!`
  - 或 footer 使用 `BREAKING CHANGE:` / `BREAKING-CHANGE:`

override 示例：

```bash
OMNE_ALLOW_MAJOR_VERSION_BUMP=1 git commit ...
```

```powershell
$env:OMNE_ALLOW_MAJOR_VERSION_BUMP = '1'
git commit ...
```

这条规则当前只根据 manifest 的版本号变化判断，不自动推断运行时兼容性。
